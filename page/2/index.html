线程基础（二）

1. 理解线程上下文切换

在多线程编程中，线程个数一般要大于CPU 个数，而每个CPU 同一时刻只能被一个线程使用。为了让用户感觉多个线程是在同时执行的，CPU 资源的分配采用了时间片轮询的策略。

也就是说，给每个线程分配一个时间片，线程在时间片内占用CPU 执行任务，当前线程使用完时间片后，就会处于就绪状态并让出CPU 让其他线程占用，这就是上下文切换

线程上下文切换的时机：当前线程的CPU时间片使用完处于就绪状态时，当前线程被其他线程中断时

1. 什么是线程

线程本身不会独立存在，它是进程中的一个实体。

进程：是系统进行资源分配和调度的基本单位，是代码在数据集合上的一次运行活动。

线程：是进程的一个执行路径，是CPU 分配的基本单位。一个进程中至少有一个线程，多个线程共享进程的资源

例;在Java中启动main 函数时，其实是启用了一个JVM 进程，而main函数所在的线程就是这个这个进程中的一个线程，也称主线程、

多个线程共享进程的堆和方法区资源，但是各个线程有自己的程序计数器和栈区域。

CQRS 与 Event Sourcing

CQRS (Command Query Responsibility Segregation), 命令查询职责分离。CQRS 是一种简单的模式。它将对象的操作分为命令与查询。查询返回数据，不会更改对象的状态。命令更改对象的状态，但不返回任何的数据。

Event Sourcing (ES, 事件回溯)， 按照 Event Sourcing 的思想， 应用不保存对象的最新状态，而是保存对象产生的所有事件，并且通过事件溯源得到对象的最新状态。

Spring 事务隔离级别

Spring 事务隔离级别

事务

事务逻辑上的一组操作，组成这组操作的各个逻辑单元，要么一起成功，要么一起失败。

事务特性 ACID

  • 原子性（Atomicity）：不可分割，事务是不可分割的最小工作单元，事务内的操作要么全做，要么全不对。
  • 一致性（Consistency）：事务的执行前后数据的完整性保持一致。
  • 隔离性（Isolation）：并发事务执行之间互不影响，在一个事务内部的操作对其他事务是不产生影响的。
  • 持久性（Durability）：事务一旦执行成功，它对数据库的改变必须是永久的

策略模式

策略模式

定义

将不同业务规则封装成不同算法，并且它们可以相互替换，实现业务逻辑与调用方逻辑的解耦合。

应用场景

  1. 统一操作有不同业务处理
  2. 业务处理逻辑需要动态切换
  3. 对象有多个行为，需要通过多重条件判断来确定具体行为
