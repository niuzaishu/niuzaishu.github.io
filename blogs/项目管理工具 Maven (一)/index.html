    • 概念

    简单来说：Maven 是一个软件项目管理和自动构建工具。

    Maven是一个项目管理工具，它包含了一个项目对象模型 (Project Object Model)，一组标准集合，一个项目生命周期(Project Lifecycle)，一个依赖管理系统(Dependency Management System)，和用来运行定义在生命周期阶段(phase)中插件(plugin)目标(goal)的逻辑。

    仓库

    构件：在 Maven 中，任何一个依赖、插件或者项目构建的输出，都可以称之为构件。

    仓库：Maven 仓库是项目中依赖的第三方库所在的位置。

    • 本地 (Local)
    • 中央 (Central)
    • 远程 (Remote)
    本地仓库

    Maven 本地存储库是一个在本地计算机上的一个文件夹位置。当你第一次运行 maven 命令的时候它就被创建了。运行 Maven 的时候，Maven 所需要的任何构件都是直接从本地仓库获取的。

    中央仓库

    Maven中央存储库是由Maven社区提供的资源库。它包含了大量的常用程序库。

    地址：http://repo1.maven.org/maven2/

    远程仓库

    这里的远程仓库主要是指”私服”，是由开发人员自己定制仓库，用来代理中央仓库及部署第三方构件。有了私服之后，当 Maven 需要下载构件时，直接请求私服，私服上存在则下载到本地仓库；否则，私服请求中央仓库，将构件下载到私服，再提供给本地仓库下载。

    也有文章认为中央仓库和”私服” 都属于远程仓库，这里这样划分主要是为了突出私服的概念，无伤大雅。

    依赖机制

    Maven 坐标：一种用于找到 Maven 依赖包的标识， 依赖引用的目标地址。

    • groupId：参考上文
    • artifactId：参考上文
    • packaging：Maven项目的打包方式， 对应依赖引用的type
    • classifier: 用来帮助定义输出一些附属文件，不能直接定义项目的classifier，因为附属输出文件不是项目直接默认生成的，而是由附加的插件帮助生成的。

    依赖引用最小信息集是{groupId, artifactId, type, classfier}。许多情况下，依赖指向的jar不需要指定classfier。因为默认type是jar，默认classfiler为空，所以我们可以把信息集设置为{groupId, artifactId}。

    Maven 遵循传递性依赖（Transitive Dependencies）：

    如果在工程A中配置依赖于B，B又依赖于C，C又依赖于D …… ，那么，Maven会将B、C、D …… 这些工程都作为A的依赖，不受解析层级的限制。

    因为依赖可传递，一个项目中的依赖可能会非常多，所以需要对依赖的传递做一些规则限制：

    • 依赖仲裁（Dependency mediation）：最短路径 & 第一声明优先
    • 依赖范围（Dependency scope）：只在特定的生命周期才引入依赖
    • 排除依赖（Excluded dependencies）：使用exclusions 排除依赖
    • 可选依赖（Optional dependencies）：使用optional定义可选依赖

    在出现传递性依赖或者没有指定版本时，也可以通过依赖管理直接指定模块版本。

    最短路径（nearest definition）

    • A -> B -> C -> D 2.0
    • A -> E -> D 1.0

    A依赖于D 1.0

    第一声明优先（the first declaration wins）

    • A -> E -> D 2.0
    • A -> E -> D 1.0

    A依赖于D 2.0

    依赖范围（Dependency scope）

    依赖范围会影响传递性依赖，同时也会影响项目构建任务中使用的classpath。

    classpath: 编译时的classpath, 测试时的classpath 和 运行时的classpath

    范围 描述 实例
    compile 默认值，编译、测试、运行三种classpath都有效 spring-core
    provided 编译、测试classpath有效，运行classpath无效，表示依赖将由JDK或者运行时的Web服务器或容器提供。编译和测试项目的时候需要该依赖，但在运行项目的时候，由于容器以及提供，就不需要Maven重复地引入一遍 servlet-api
    runtime 测试和运行classpath有效，编译classpath无效。编译只需要JDK提供的JDBC接口，只有在执行测试或者运行项目的时候才需要实现上述接口的具体JDBC驱动。 JDBC驱动
    test 测试classpath有效，编译和运行classpath无效。 JUnit
    system 与provided的范围相同，通过systemPath元素显示地指定依赖文件的路径 本地的，不在仓库中的包。
    import 不针对任何一种classpath，用于导入其他pom中的dependencyManagement元素

    除了import 外的依赖范围，在依赖传递时会有不同影响。如下表。最左侧一列代表了直接依赖范围，最顶层一行代表了传递性依赖的范围，行与列的交叉单元格就表示最终的传递性依赖范围。表中的“-“表示该传递性依赖将会被忽略。

    compile provided runtime Test
    compile compile - runtime -
    provided provided - provided -
    runtime runtime - runtime -
    test test - test -

    POM

    Maven项目使用项目对象模型（Project Object Model，POM）来配置，项目对象模型存储在名为 pom.xml 的文件中。

    Super POM

    Super POM是 Maven 默认的 POM。所有的 POM 都继承自一个Super POM（无论是否显式定义了这个Super POM）。Super POM 包含了一些可以被继承的默认设置。
